## 一、开启 Django 服务

每次要访问网页前，在终端执行：

```powershell
cd d:\SoftWare\Code\bysj
python manage.py runserver
```

看到 `Starting development server at http://127.0.0.1:8000/` 后，在浏览器打开：**http://127.0.0.1:8000/**

- 使用 **MySQL**：请先启动 MySQL 服务，再执行上述命令。
- 使用 **SQLite**（未装 MySQL 时）：先执行 `$env:USE_SQLITE="1"`，再执行 `python manage.py runserver`。

## 五、常用测试流程

1. **用户管理员**：user_admin 登录 → 用户管理、入组审核
2. **药品管理员**：med_admin 登录 → 标准药库、药箱药品审核、帖子药品审核
3. **帖子管理员**：post_admin 登录 → 帖子审核 → 通过待审帖
4. **家庭组**：张家爸/李家爸 登录 → 申请加入家庭组（待用户管理员审核）
5. **共享转让**：张家妈/李家妈 登录 → 共享广场 → 发帖/留言 → 药品审核通过后可加入家庭药箱
6. **服药提醒**：普通用户登录 → 家庭药箱新增/编辑药品时开启提醒 → 进入“服药提醒”页面查看到点药品
7. **每日医学小贴士**：sys_admin 登录 → 发布医学小贴士（建议每日 1 条）→ 所有用户在“用户信箱/医学小贴士”查看
8. **用户信箱**：普通用户登录 → 消息中心查看私聊、帖子互动、公告、小贴士的统一聚合

## 六、创建命令

```bash
cd d:\SoftWare\Code\bysj
python manage.py create_test_users      # 创建管理员账号
python manage.py create_family_and_posts # 创建两组家庭、药品、帖子
```

## 八、双审核机制（串行）

共享帖子采用串行双审核，避免“帖子还没过，药品先过”的体验不一致：

1. 普通用户发帖后，状态为：
   - 帖子审核：待审核
   - 药品审核：待帖子审核
2. 帖子管理员审核通过后，药品审核状态自动变更为“待药品审核”
3. 药品管理员审核通过后，帖子才会进入共享广场，且可“加入家庭药箱”
