# 💊 基于 Django 的家庭医药管理与共享平台

## 📖 项目简介
本项目是一个基于 B/S 架构的家庭医药协作与共享系统（亦为本科毕业设计）。旨在解决家庭药品管理混乱、过期浪费等问题，并创新性地构建了基于信任机制的跨家庭闲置药品互助渠道，同时支持家庭内部的用药提醒与健康协同记录。

## 🛠️ 技术栈
- **后端开发**: Python 3.x, Django 5.2
- **数据库存储**: MySQL / SQLite（可选，见 `数据库切换说明.md`）
- **前端界面**: HTML / CSS / JavaScript, Bootstrap 5
- **运行与依赖**: `requirements.txt`（Django、PyMySQL 等）
- **开发与协作**: VS Code / Cursor, Git, Navicat

## ✨ 核心业务模块
1. **多角色权限隔离**  
   基于 RBAC 思想，实现超级管理员、家庭管理员与普通成员的精细化权限边界控制（如家庭成员管理、药品审核、公告发布等）。
2. **家庭药箱智能管理**  
   支持药品的增删改查、效期临近监控、库存阈值报警，以及按家庭维度的用药记录与提醒。
3. **跨家庭药品互助审批流**  
   打造完整的「发布 → 申请 → 审批 → 流转 → 完成」状态机闭环，记录每一步操作轨迹，便于追踪与审计。

## 🔥 核心技术亮点
- **高并发防超卖机制**  
  在处理跨家庭的闲置药品互助申请时，为保证数据绝对一致性，采用 Django 的 `select_for_update()` 结合 MySQL **行级悲观锁**，并配合 `transaction.atomic()` **原子事务**，有效避免高并发场景下的药品超卖问题。
- **数据强一致性设计**  
  数据库底层设计遵循第三范式，合理利用外键约束、唯一约束与事务回滚机制，确保库存扣减、审批记录、多表写入等操作的原子性与一致性。
- **安全性与防越权控制**  
  通过自定义装饰器与中间件对关键接口做鉴权与角色校验，结合 Django 路由与模板层判断，有效防止未登录访问与跨家庭的水平越权。

## 🚀 快速开始

```bash
# 1. 克隆项目
git clone https://github.com/aotemanbingyue/family-medicine-system.git
cd family-medicine-system

# 2. 安装依赖
pip install -r requirements.txt

# 3. 根据需要选择数据库，并配置环境变量
#    仅本地快速体验可直接使用 SQLite：
#    PowerShell:
#    $env:USE_SQLITE="1"

# 4. 生成并迁移数据库
python manage.py migrate

# 5. 启动开发服务器
python manage.py runserver
```

更详细的运行说明见 **新手入门指南.md** 与 **Django服务说明.md**。
