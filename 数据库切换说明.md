# 数据库切换说明（含数据备份，避免丢失）

项目支持 **MySQL** 与 **SQLite** 两种数据库，可按需切换。**切换前请先备份数据**，再按下面步骤操作。

---

## 一、切换前：先备份当前数据库的数据

在切换数据库前，用 Django 的 `dumpdata` 导出当前库里的数据，以免丢失。

### 1. 确保当前数据库能正常使用

- 若当前用的是 **SQLite**：无需额外配置，直接执行下面命令即可。
- 若当前用的是 **MySQL**：请先启动 MySQL 服务，确保能正常连接。

### 2. 执行备份命令（PowerShell）

```powershell
cd d:\SoftWare\Code\bysj
python manage.py dumpdata app auth.User auth.Group auth.Permission contenttypes sessions --natural-foreign --natural-primary --indent 2 -o backup_data.json
```

- `backup_data.json` 会保存在项目根目录。
- 包含：用户、药品、帖子、留言、公告、会话等应用数据。

### 3. 确认备份文件已生成

检查项目目录下是否有 `backup_data.json`，且文件大小不为 0。

---

## 二、切换到 SQLite（无需 MySQL）

### 1. 设置环境变量并启动

```powershell
cd d:\SoftWare\Code\bysj
$env:USE_SQLITE="1"
python manage.py migrate
python manage.py runserver
```

- 首次切换到 SQLite 时会自动创建 `db.sqlite3`。
- 数据保存在项目根目录的 `db.sqlite3` 文件中。

### 2. 若之前有备份，可导入数据

```powershell
python manage.py loaddata backup_data.json
```

---

## 三、切换到 MySQL

### 1. 准备工作

- 启动 MySQL 服务。
- 创建数据库：
  ```sql
  CREATE DATABASE bysj CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
  ```
- 若 root 密码不是 123456，先设置：
  ```powershell
  $env:DJANGO_DB_PASSWORD="你的MySQL密码"
  ```

### 2. 取消 SQLite 并迁移

```powershell
cd d:\SoftWare\Code\bysj
# 若之前设过 USE_SQLITE，先取消
$env:USE_SQLITE=""
python manage.py migrate
python manage.py runserver
```

### 3. 若之前有备份，可导入数据

```powershell
python manage.py loaddata backup_data.json
```

---

## 四、完整切换流程示例（SQLite → MySQL，并保留数据）

```powershell
cd d:\SoftWare\Code\bysj

# 1. 当前使用 SQLite，先备份
$env:USE_SQLITE="1"
python manage.py dumpdata app auth.User auth.Group auth.Permission contenttypes sessions --natural-foreign --natural-primary --indent 2 -o backup_data.json

# 2. 取消 SQLite，改用 MySQL（需已启动 MySQL 并建好 bysj 库）
$env:USE_SQLITE=""

# 3. 在 MySQL 中创建表结构
python manage.py migrate

# 4. 导入备份数据
python manage.py loaddata backup_data.json

# 5. 启动服务
python manage.py runserver
```

---

## 五、常见问题

- **loaddata 报错或冲突**：若提示主键冲突，可尝试只导入 app 数据：
  ```powershell
  python manage.py dumpdata app --indent 2 -o app_backup.json
  # 切换数据库并 migrate 后
  python manage.py loaddata app_backup.json
  ```
- **MySQL 连接失败**：检查 MySQL 是否已启动，数据库是否已创建，密码是否正确。
