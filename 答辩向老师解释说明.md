# 答辩向老师解释说明

> 用于毕业论文答辩时向导师/老师讲解系统设计与实现。

---

## 一、项目概述（开场白）

**老师好，我设计的系统叫「家庭药品管理系统」，主要面向家庭用户，解决家里药品过期浪费、信息散乱、无法协作等问题。**

核心功能包括：

1. **家庭药箱**：录入家里的药品，记录生产/过期日期，支持过期提醒  
2. **家庭组协作**：一家人共享药箱信息，避免重复囤药  
3. **共享广场**：用户可发布闲置药品转让，其他用户可浏览、留言联系  
4. **全局药品库**：管理员维护标准药品字典，普通用户可查询  
5. **角色权限**：药品管理员、帖子管理员、系统管理员各司其职  

---

## 二、技术架构（老师常问）

| 层次 | 技术选型 | 说明 |
|------|----------|------|
| 后端 | Django 5.2 + Python 3.11 | MTV 架构，ORM 操作数据库 |
| 数据库 | MySQL（默认）/ SQLite | 支持切换，便于开发与部署 |
| 前端 | Bootstrap 5 + Vue 3 | 响应式 UI，部分页面用 Vue 做交互 |
| 认证 | Django 自带 Auth | 扩展 AbstractUser，自定义角色字段 |

## 三、如何讲「全局药品库」

**话术示例：**

> 「全局药品库是本系统的一个核心模块。它由药品管理员维护，相当于一个标准药品字典，包含药品通用名、分类、条形码、功效说明等。  
> 普通用户只能查询，不能修改。用户在添加家庭药箱时，可以关联这个标准库，保证数据规范性。  
> 药品管理员和系统管理员可以增删改查这个库，但药品管理员不能审核帖子、不能发公告，权限是严格分离的。」

---

## 四、如何讲「角色与权限分离」

| 角色 | 能做的事 | 不能做的事 |
|------|----------|------------|
| 普通用户 | 家庭药箱、过期提醒、家庭组、共享转让、药品查询 | 不能管理标准药库、不能审核帖子、不能发公告 |
| 药品管理员 | **仅** 标准药库的增删改查 | 不能审核帖子、不能发公告 |
| 帖子管理员 | **仅** 审核共享帖子 | 不能管药库、不能发公告 |
| 系统管理员 | 药库 + 帖子审核 + 发公告（全部） | - |

**话术示例：**

> 「系统采用 RBAC 思路，不同管理员职责分离。药品管理员只能管药品，帖子管理员只能管帖子，这样既符合实际业务，也便于安全审计。」

---

## 五、核心难点与解决方案（论文/答辩亮点）

1. **软删除**：药品、帖子等用 `is_deleted` 标记，不真删数据，可追溯、可恢复  
2. **家庭组**：通过 `family_id` 关联一家人，实现家庭成员药箱共享  
3. **帖子审核**：发布后待审核，通过才在广场展示，避免违规内容  
4. **消息提醒**：留言与回复有未读红点，便于用户及时响应  

---

## 六、演示流程建议（给老师看）

1. 首页 → 展示登录后卡片式入口  
2. 家庭药箱 → 添加药品、编辑、过期提醒  
3. 药品查询 → 展示全局药品库（只读）  
4. 共享广场 → 发帖、留言、审核（若为管理员）  
5. 管理后台 → 用药品管理员账号演示「只能管药库」  

---

## 七、常见追问及回答

- **为什么用 Django？** 成熟稳定，自带 Admin、Auth、ORM，适合快速开发毕业设计。  
- **数据库为什么支持切换？** 开发时可用 SQLite 免装 MySQL，部署时用 MySQL 保证性能。  
- **前后端是分离的吗？** 目前是 Django 模板 + 少量 Vue，属于传统架构；后续可扩展为 Django REST + Vue SPA。
