# Django 毕业设计 · 新手入门指南

按 `.cursorrules` 的约定，这里用「管理员登录后才能查看药品列表」为例，分步说明 Django 开发流程。

---

## 一、先搞清这几个文件是干什么的

| 文件 | 作用 |
|------|------|
| `app/models.py` | 定义数据表（用户、药品、帖子等） |
| `app/views.py` | 处理请求、查数据库、返回页面 |
| `app/urls.py` | 把网址和视图对应起来 |
| `app/templates/app/` | 前端页面模板（HTML） |
| `app/forms.py` | 表单验证 |
| `app/decorators.py` | 权限装饰器（如：只有管理员能进） |

**记住顺序**：用户访问网址 → `urls.py` 找到对应视图 → `views.py` 做逻辑 → 用 `templates` 里的模板渲染页面。

## 二、示例：管理员登录后才能查看药品列表

### 第一步：在 `app/views.py` 里加上登录/权限限制

在要限制的视图函数上加上 `@login_required` 或 `@role_required`：

```python
# app/views.py 中，在某个视图函数上方添加装饰器

from django.contrib.auth.decorators import login_required
from .decorators import role_required

# 仅登录用户可访问（示例）
@login_required
def family_medicine_list(request):
    ...

# 仅药品管理员或系统管理员可访问（示例）
@login_required
@role_required('admin_med', 'admin_sys')
def global_medicine_list(request):
    ...
```

- `@login_required`：必须登录，否则跳转到登录页。
- `@role_required('admin_med', 'admin_sys')`：必须是这些角色之一，否则提示无权限并跳转。

你的项目里 `global_medicine_list` 已经这样写了，所以「标准药库」本身就是「管理员登录后才能查看」的。

## 三、如果你想自己做一遍：从零加一个「管理员专属页面」

### 第一步：在 `app/views.py` 里新增一个视图

在文件末尾添加（保留原有缩进风格）：

```python
# ---------- 示例：管理员专属的药品统计页 ----------
@login_required
@role_required('admin_med', 'admin_sys')  # 仅药品管理员、系统管理员可访问
def admin_medicine_stats(request):
    """
    管理员专属：药品统计页面。
    用于答辩时演示「按角色限制访问」。
    """
    from .models import GlobalMedicine, FamilyMedicine
    total_global = GlobalMedicine.objects.filter(is_deleted=False).count()
    total_family = FamilyMedicine.objects.filter(is_deleted=False).count()
    return render(request, 'app/admin_medicine_stats.html', {
        'total_global': total_global,
        'total_family': total_family,
    })
```

### 第二步：在 `app/urls.py` 里加路由

在 `urlpatterns` 中增加一行（位置随意，建议放在其他 admin-med 路由附近）：

```python
path('admin-med/stats/', views.admin_medicine_stats, name='admin_medicine_stats'),
```

### 第三步：新建模板 `app/templates/app/admin_medicine_stats.html`

```html
{% extends 'app/base.html' %}
{% block title %}药品统计 - 药品管理系统{% endblock %}
{% block content %}
<h2 class="mb-4">药品统计（管理员专属）</h2>
<div class="card">
    <div class="card-body">
        <p>全局标准药库：<strong>{{ total_global }}</strong> 种</p>
        <p>家庭药箱记录：<strong>{{ total_family }}</strong> 条</p>
    </div>
</div>
{% endblock %}
```

### 第四步：在导航栏加上入口（可选）

在 `app/templates/app/base.html` 中，药品管理员相关的菜单里加一条：

```html
<li class="nav-item"><a class="nav-link" href="{% url 'admin_medicine_stats' %}">药品统计</a></li>
```

### 第五步：运行并验证

```powershell

python manage.py runserver
```

用 `med_admin` 登录，访问 `/admin-med/stats/`，应能打开；用 `user1` 登录访问，应提示无权限。

## 四、常用终端命令（方便复制）

```powershell

python manage.py migrate

# 检查配置是否有问题
python manage.py check
```

---

## 五、开发新功能时的推荐顺序

1. **改 `models.py`**（如果要新表或新字段）
2. **执行迁移**：`makemigrations` → `migrate`
3. **改 `views.py`**：写视图逻辑，加装饰器做权限
4. **改 `urls.py`**：加路由
5. **建模板**：在 `app/templates/app/` 下新建 HTML
6. **运行 `runserver` 测试**

按这个顺序做，一般不会乱。
