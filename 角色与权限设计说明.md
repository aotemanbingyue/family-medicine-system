# 角色与权限设计说明

> 解答：「药品管理员能不能管别的？」——**不能，权限是按功能严格分离的。**

---

## 一、角色与功能对应表

| 角色代码 | 显示名称 | 可见功能 | 不可见/不可访问 |
|----------|----------|----------|------------------|
| `user` | 普通家庭用户 | 家庭药箱、过期提醒、家庭组、共享广场、药品查询、系统公告 | 标准药库维护、帖子审核、发布公告 |
| `admin_med` | 药品管理员 | 上述全部 + **标准药库**（增删改查） | 帖子审核、发布公告 |
| `admin_post` | 帖子管理员 | 上述全部 + **帖子审核** | 标准药库维护、发布公告 |
| `admin_sys` | 系统管理员 | 上述全部 + **发布公告** | 无（拥有所有权限） |

---

## 二、实现方式

### 1. 视图层：`@role_required` 装饰器

```python
# 药品管理员 或 系统管理员 才能访问
@login_required
@role_required('admin_med', 'admin_sys')
def global_medicine_list(request):
    ...

# 帖子管理员 或 系统管理员 才能访问
@role_required('admin_post', 'admin_sys')
def post_audit_list(request):
    ...

# 仅系统管理员 才能访问
@role_required('admin_sys')
def announcement_create(request):
    ...
```

- **药品管理员** 只出现在 `global_medicine_*` 的 `allowed_roles` 里  
- **帖子管理员** 只出现在 `post_audit_*` 的 `allowed_roles` 里  
- **系统管理员** 出现在所有管理员接口里  

### 2. 导航栏：按角色控制显示

```django
{% if user.role == 'admin_med' or user.role == 'admin_sys' %}
<li><a href="...">标准药库</a></li>
{% endif %}
{% if user.role == 'admin_post' or user.role == 'admin_sys' %}
<li><a href="...">帖子审核</a></li>
{% endif %}
{% if user.role == 'admin_sys' %}
<li><a href="...">发布公告</a></li>
{% endif %}
```

- 药品管理员：只看到「标准药库」，看不到「帖子审核」「发布公告」  
- 帖子管理员：只看到「帖子审核」，看不到「标准药库」「发布公告」  
- 系统管理员：三个都看到  

### 3. URL 直接访问也会被拦截

即使知道 `/admin-post/audit/` 的地址，**药品管理员** 访问也会被 `@role_required` 重定向并提示「您没有权限访问该功能」。

---

## 三、可能产生的混淆

| 现象 | 原因 | 说明 |
|------|------|------|
| 看到能管药又能审帖子 | 登录的是 **系统管理员** | 系统管理员本身设计为最高权限 |
| Django 自带的 /admin/ | `is_staff` 用户可进 | 这是 Django Admin，和本系统的角色是两套；建议答辩时不给老师看 /admin/，只演示自定义的管理页面 |
| 药品管理员能查药品 | 药品查询 `/medicine/search/` 对所有人开放 | 这是**只读查询**，不是维护药库，符合业务 |

---

## 四、总结

**药品管理员只管理药品（标准药库），不能管理帖子审核和系统公告。**  
视图层和导航层都做了限制，权限是按功能清晰分离的。
